Playing Vim - Vim für Fortgeschrittene
Die Leser des Linux Journals haben beim Linux Journal Readers' Choice Awards 2005, 2004, 2003, 2002 und 2001 den Vim in der Kategorie Texteditor zum beliebtesten seiner Art gewählt. Seine Beliebtheit beruht auf seinem effizienten UI- und Editier-Konzept, dass einem hervorragend erlaubt, ASCII-Dateien schnell und ökonomisch zu bearbeiten. Sicher, diese Effizienzmöglichkeit bedeutet für den User eine steile Lernkurve und nicht jeder mag zuerst in einen Lernaufwand investieren um später mit mehr Freude Editiertätigkeiten an Sourcecode-, Text- und Konfigurationsdateien vornehmen zu können.
"Playing Vim" richtet sich an diejenigen, die den Erstkontakt mit Vim überlebt und sich nach dem Schock sogar weiter mit dem Vim beschäftigt haben. Sie finden hier Ideen und Anregungen, wie der Editor genutzt werden kann.
Für diejenigen, die sich noch nicht so weit fühlen, lohnt sich ein Blick auf "Surviving Vim".
"Playing Vim" wird zum Warmwerden mit einer kurze Wiederholung des Basics - wie in "Surviving Vim" beschrieben - beginnen. Danach geht es in die Tiefen der Möglichkeiten des Vim Texte schneller und effizienter zu edizieren.

Schnelles (Weiter)Suchen
Ohne viel Tipperei ein Wort suchen.
Im Command Mode kann man nach dem Wort unter dem Cursor mit * vorwärts suchen und mit # rückwärts suchen. Mit [n] wird in die selbe Richtung weiter gesucht und mit [N] in die entgegengesetzte Richtung.
[*] -> vorwärts suchen nach Wort unter dem Cursor
[#] -> rückwärts suchen nach Wort unter dem Cursor
[n] -> in die selbe Richtung weiter suchen
[N] -> in die entgegengesetzte Richtung weitersuchen

Tabs und Spaces und Trailling Spaces
Tabs oder Spaces, was soll ich nehmen, das ist hier die Frage (die hier nicht beantwortet wird).
see the difference between tabs and spaces and trailing blanks

:set listchars=tab:>-,trail:-,eol:<
oder :set listchars=tab:»·,trail:· ([ALTGR]-[x] -> » [ALTGR]-[.] -> ·
:set (no)list

no tabs, only spaces (tab is 4 spaces)

:set tabstop=4 -> tab is 4 chars long
:set shiftwidth=4 -> number of space chars inserted for indentation
:set expandtab -> space instead of tab
:set softtabstop=4 -> makes the tab-spaces feel like real tabs
:retab -> convert exisiting tabs in spaces
~/.vimrc -> au FileType make setlocal noexpandtab -> turn off expandtab for editing makefiles

Substitute
Mit Hilfe Regulärer Ausdrücke können Ersetzungen vorgenommen werden. Ein kleiner Einblick in Regexp.
:[range]s[ubstitute]/pattern/replace/[g][c][i][I]

[range] -> [addr1[,addr2]] -> wenn nicht angegeben, wird :s auf die aktuelle Zeile angewandt
- addr1 -> auf dieser Zeile wird : angewendet
- addr1,addr1 -> im Bereich beider Zeilen wird :s angewendet
- % -> auf die ganze Datei wird :s angewendet, wie 1,$
pattern -> nach was gesucht wird
- im einfachsten Fall wie geschrieben, z.B. alt sucht nach alt, halt, alte u.s.w.
- besteht aus qualifieer und quantifieer
- qualifieer: wie a oder abc oder [a-z] oder . (jedes Zeichen)
- quantifieer: wie * (0<=), + (0<), ? (0 oder 1), = (0 oder 1), {n,m} (n bis m), {n} (genau n), {n,} (mindestens n), {,m} (0 bis m), u.s.w
- hold buffer, der über (pattern) definiert und über n, wobei (n=1-9) ist, referenziert wird
- | alternation
- ^ start of line
- $ end of line
- < start of word (escape with backslash)
- > end of word (escape with backslash)
replace -> mit was ersetzt wird
- im einfachsten Fall wie geschrieben
- hold buffer aus pattern
- & replaced by pattern match
g -> global, alle Vorkommen einer Zeile
c -> confirm, vor jeder Ersetzung wird um Bestätigung gebeten
i -> ignore case for pattern
I -> respect case for pattern

Beispiele

:%/s*$// -> trim end-of-line tabs and spaces

Swapping of Characters or Lines
Zeichen oder Zeilen zu vertauschen ist nur ein paar Befehle entfernt.
Getippt und zwei Zeichen vertauscht? Mit [x]+[p] lassen sich zwei Zeichen vertauschen.
Die Reihenfolge der Zeilen stimmt nicht? Mit [d]+[d]+[p] sind zwei Zeilen vertauscht.
[x] -> löscht Zeichen unter dem Cursor
[p] -> fügt Zeichen hinter dem Cursor ein
[d]+[d] -> löscht Zeile
[p] fügt Zeile unter dem Cursor ein
[x]+[p] -> swap two characters
[d]+[d]+[p] -> swap two lines

Deletion Buffer
In Vim kann man auf die letzten neun Löschungen zugreifen.
Wenn etwas gelöscht wird, wird es von Vim im Deletion Buffer vorgehalten. Dieser Deletion Buffer ist neun Einträge groß. Mit "np (n=1-9) kann man auf diese Deletion Buffer zugreifen.
"2p -> einfügen der vorletzten Löschung hinter oder unter der Cursorposition
"1pu.u.u.u.u.u.u.u.u -> step through deletion buffer
top 
 
Copy Buffer
Wenn man etwas kopieren möchte, kann man es zur späteren Verwendung in einen benannten Buffer legen.
Wenn man etwas editiert, fügt man auch immer wieder das selbe ein. Man kann in Vim mehrere Copy Buffer nutzen, um verschiedene Sachen zu kopieren und nach Bedarf einfügen. Der Copy Buffer ist 26 Einträge groß. Mit "CHARy (CHAR=a-z) wird das Markierte in den Copy Buffer CHAR gelegt und mit "CHARp wird es hinter/unter dem Cursor eingefügt.
"hy -> markiertes wird in den Copy Buffer h gelegt
"hp -> das sich im Copy Buffer h befindende wird hinter/unter dem Cursor eingefügt
top 
 
Option handling
Neben der Möglichkeit Optionen über die ~/.vimrc und Modelines zu verwalten, könne sie auch über die Ex Command Line gesetzt und abgefragt werden.
Die Ex Command Line erreicht man über [:] im Command Modus. Mit :set wird die Abfrage oder das Setzen der Option eingeleitet.
:set OPTION -> setzen der Option OPTION
:set noOPTION -> das Nichtsetzen einer Option
:set OPTION! -> toggeln der Option
:set OPTION? -> Abfragen einer Option
:set all? -> Abfragen aller gesetzter Optionen
:set all& -> set all options to their default value
:set -> Abfragen aller Optionen, die vom Default-Wert abweichen
top 
 
Howto write a file
Es gibt einige Wege, wie man das, was im Buffer ist, auf die Festplatte bringen kann.
:w -> schreibt den Buffer in die geöffnete Datei
:w NEWFILE -> schreibt den Buffer in eine neue Datei mit Namen NEWFILE
:15,42w NEWFILE -> schreibt die Zeilen 15 bis 42 in die (neue) Datei NEWFILE
:w >> OTHERFILE -> hängt den Buffer an die Datei OTHERFILE an
:15,42w >> OTHERFILE -> hängt die Zeilen 15 bis 42 an die Datei OTHERFILE an
:x -> schreibt bei modifizierten Buffer und beendet Vim (bei nicht-modifiziertem Buffer wird die c-time nicht geändert)
ZZ -> wie :x

Drucken
Manchmal möchte man etwas ausdrucken.
:hardcopy -> Vim übergibt das erstellte PostScript lpr
:w !lp -> Der Buffer wird als Stream lp übergeben (:w! lp or :w!lp -> force writing of file lp)
:10,20w !lp -> die Zeilen 10 bis 20 werden lp übergeben
:10,20hardcopy -> für die Zeilen 10 bis 20 wird ein PostScript erstellt und lpr übergeben

Howto open a file
Verschieden Wege eine Datei zu öffnen.
$ vim FILE -> der normale Weg FILE zu öffnen
$ vim + FILE -> öffnet FILE in der letzten Zeile
$ vim +n FILE -> öffnet FILE in Zeile n
$ vim /PATTERN FILE -> öffnet FILE beim ersten Vorkommen von PATTERN
:e[edit] FILE -> öffnet FILE bei geöffnetem Vim
:sp[lit] FILE -> öffnet FILE und splittet den Screen horizontal, um die schon geöffnete Datei weiter anzuzeigen
:vs[plit] FILE -> öffnet FILE und splittet den Screen vertikal, um die schon geöffnete Datei weiter anzuzeigen

Range
Viele EX-Kommandos lassen sich über Ranges steuern. Hier eine kleine Zusammenfassung.
:n,md -> delete from line n to m
:23,45d -> die Zeilen 23 bis 45 werden gelöscht
0 -> vor erster Zeile
1 -> erste Zeile
$ -> letzte Zeile
. -> aktuelle Zeile
-n -> von aktueller Zeile n zurück
+m -> von aktueller Zeile m weiter
:n,+m -> von Zeile n und m Zeilen weiter
:1,$ -> der ganze Buffer
:% -> der ganze Buffer

Movement und Textobjects
Verschiedenen Command Mode Befehlen sind Textobjekte als Operationsziel mitzugeben. Die Textobjekte sind gleichzeitig Movementbefehle im Command Mode.
w -> word; eg dw -> delete word
) -> end of sentence; d) -> delete to end of sentence
} -> end of paragraph; d} -> delete to end of paragraph
j -> one line down ; dj -> delete current and next line

Visual Mode
Command Mode und Ex Mode Befehle, denen ein Bereich übergeben werden kann, können dies auch über den Visual Mode bekommen.
v -> Visual Mode
V -> Visual Mode linewise
[STRG]-[v] -> Visual Block Mode
Mit v, V oder [STRG]-v in den Visual Mode den Bereich markieren, auf den der Befehl angewendet werden soll. Danach den gewünschten Befehl eingeben.

Modes
Vim ist ein modaler Editor.
Normal Mode (Command Mode)

Vim startet im Normal Mode.
Mit [ESC] kommt man zurück in den Normal Mode.

Insert Mode

Der Insert Mode erlaubt das Eingeben von Text.
Mit i, I, a, A, c, o, O, r kommt man in den Insert Mode.

Ex Mode (Last Line Mode)

Ex Mode Kommandos werden über [:] eingeleitet und mit [ENTER] abgeschlossen.

Visual Mode

Der Visual Mode dient dem Markieren von Textteilen.
Mit [v] startet der Character-orientierte Visual Mode. Mit [V] startet der zeilenorientierte Visual Mode. Mit [CTRL]-[v] startet der blockorientierte Visual Mode.

Persistenz
Mit Hilfe der 'history' und 'viminfo' Konfigurationsdirektive kann man die Informationen einer Vim-Session zur Nächsten übertragen.
Vim hält eine History seiner Command Line vor, wenn :set history= größer Null ist. Für die Suche wie die Ex-Style Kommandos werden separate History-Tables vorgehalten.
In der ~/.vimrc wird die Größe der History gesetzt. (default=20)
set history=50

Innerhalb von Vim greift man auf die History der Ex-Style Kommandos zu mit
:history

und auf die History der Search Strings mit
:history /


Zur Persistenz verschiedener Informationen wird die viminfo Direktive genutzt.
Über die damit gesteuerte ~/.viminfo Datei lassen sich Informationen zur History der Command Line und Search Pattern speichern, Texte in den Registern (Copy und Delete Buffer), Marks für verschiedene Dateien, die Buffer Liste und globale Variablen.
Innerhalb von Vim kann man sich die einkompilierten Features anzeigen lassen, um zu sehen, ob +vimfinfo aufgeführt ist.
:version

Die zur Persistenz genutzte Datei ist defaultmäßig ~/.viminfo.
Die aktuell gesetzte viminfo Direktive lässt sich innerhalb von Vim abrufen.
:set viminfo?

Konfiguriert wird die viminfo Direktive in der ~/.vimrc mit
set viminfo=%,'20,/100,:100,<500,f1,n~/.vim/viminfo

% -> save and restore buffer list
'20 -> save marks for this many files
/100 -> number of lines from search history
:100 -> number of lines from command line history
<500 -> how many lines are saved for each register
f1 -> save global marks
n -> name of viminfo file
Die einzelnen Direktiven werden mit Komma separiert. Die n-Direktive muss an letzter Stelle stehen.

Die Informationen lassen sich zwischen zwei Vim-Sessions austauschen, indem in der einen mittels :wv per Hand die ~/.viminfo Datei geschrieben und in der anderen mittels :rv (eventuell :rv! zum Überschreiben der schon angefallenen neuen Informationen) die Informationen der ~/.viminfo eingelesen werden.

:help viminfo
:help 'viminfo'

Hilfe
Vim ist sehr gut dokumentiert und hat eine sehr gute Hilfe. Ein paar Tipps zur Bedienung.
:help BEGRIFF [ENTER]-> zeigt die erste Fundstelle zu BEGRIFF

mehrere Fundstellen finden (1)

:help BEGRIFF [STRG]-[d] -> zeigt alle Fundstellen zu Begriff
:help BEGRIFF [TAB] -> läuft alle Fundstellen durch

über die Hilfe greppen

:helpgrep BEGRIFF [ENTER] -> grept über die Dokumentation nach BEGRIFF
:cw[indow] -> öffnet ein Fenster mit allen Fundstellen von :helpgrep; mit Cursortasten durchlaufen; [ENTER] öffnet Fundstelle
:helpgrep BEGRIFFc -> grept case-insensitiv über die Dokumentation nach BEGRIFF
:help 'OPTION' -> spring zur Beschreibung der OPTION in options.txt

navigieren

[STRG]-[]] -> jumps to tag
[STRG]-[t] oder [STRG]-[o] -> jumps back

mehrere Fundstellen finden (2)

:tag /BEGRIFF -> springt zur ersten Fundstelle mit BEGRIFF
:tnext -> springt zur nächsten Fundstelle

Hilfe in einem neuen Konsolenfenster öffnen

:!xterm -e vim +help &
:!xterm -e gvim +help &
:!gvim +help &

:helpf[ind] -> shows a dialog to enter and gives a little help

Prepend i_ for Insert Mode Commands
Prepend c_ for Command Line editing Commands
Prepend ' for option name
top 
 
Buffer
In Vim wird das Bearbeitete als Buffer bezeichnet. Die geladene Datei wird erst durch das Speichern des Buffers geändert. Buffer können activ, hidden oder inactiv sein. Ein active Buffer wurde von einer Datei geladen und ist im Fenster dargestellt. Ein hidden ist eine geladene Datei, die jedoch nicht dargestellt wird. Ein inactive Buffer ist ein nicht dargestellter und leerer Buffer, dessen Optionen und Marks für die zugrunde liegende Datei über die .viminfo bekannt sind.
:buffers -> zeigt die Bufferliste an; active Buffer mit 'a' und hidden mit 'h'
:files -> zeigt die Bufferliste an; active Buffer mit 'a' und hidden mit 'h'
:ls -> zeigt die Bufferliste an; active Buffer mit 'a' und hidden mit 'h'
[STRG]-[^] -> Hin- und herspringen zw dem aktuellen und letzten Buffer
[N]-[STRG]-[^] -> zeigt Buffer mit Zahl N aus :buffers an, öffnet gegebenenfalls die Datei
:[N]buffer -> zeigt Buffer mit Zahl N aus :buffers an, öffnet gegebenenfalls die Datei
:[N]sbuffer -> splittet das Fenster und zeigt Buffer mit Zahl N aus :buffers an, öffnet gegebenenfalls die Datei
top 
 
Sonderzeichen
Sonderzeichen und andere durch die Tastatur nicht direkt eingebbare Zeichen lassen sich im Insert-Modus via [STRG]-[k] eingeben.
:digraphs -> Liste der Sonderzeichen und deren Tastenkombination
[STRG]-[k]+[DIGRAPH] -> Eingabe des Sonderzeichen aus :digraphs mit der Tastenkombination DIGRAPH
[STRG]-[k]+[u]+['] -> ú
[STRG]-[k]+[o]+[/] -> ø
[STRG]-[k]+[a]+[>] -> â

Marks - Markierungen
Vim erlaubt das Setzen von Marken. Zu diesen Marken kann gesprungen werden. Sie können weiter als Objekte oder Adressen für Operationen genutzt werden. Zur dateiweiten Identifizierung werden Kleinbuchstaben [a-z] genutzt (local marks). Großbuchstaben [A-Z] markieren Marks über die Dateigrenze hinweg (global marks).
ma -> setzen des Mark a an der aktuellen Cursorposition
`a -> springt zur Stelle des Mark a
'a -> springt zum Zeilenanfang des Mark a
:'a,'b move . -> verschiebt den Text zwischen den Marks a und b an die aktuelle Stelle
mB -> setzen des global Marks B
`B -> springt zum global Mark B, öffnet die Datei gegebenenfalls und zeigt die Datei im Fenster an
top 
 
Systembuffer - Austausch zwischen Vim und anderen Programmen
Über den Systembuffer oder auch Zwischenablage kann Text zwischen Vim und anderen Programmen (oder einer anderen Vim Session) ausgetauscht werden. Der Systembuffer wird über "* erreicht.
["]+[*]+[p] -> fügt den Inhalt der Systemzwischenablage hinter / unter dem Cursor ein
["]+[*]+[y]+[3]+[w] -> kopiert die nächsten drei Wörter in die Systemzwischenablage
top 
 
Indenting
Vim unterstützt beim Neueinrücken (re-indent) von Code.
[=]+[=] -> die aktuelle Zeile wird entsprechend der Position im Code neu eingerückt (re-indent)
[=]+[a]+[{] -> der aktuelle Codeblock wird neu eingerückt (re-indent)
[g]+[g]+[=]+[G] -> der Code der aktuellen Datei wird neu eingerückt (re-indent)
top 
 
abc der Kommandos
Das kleine Alphabet der Kommandos. Siehe auch :help index.
a -> append
b -> backward by word
c -> change
d -> delete
e -> forward by word (to end of)
f -> move to character towards right (fCHAR)
g -> :help g (Commands starting with 'g')
h -> left move
i -> insert mode
j -> down move
k -> up move
l -> right move
m -> set mark{a-Z}
n -> repeat search
o -> open line under cursor position
p -> paste
q -> :help q (Complex repeats)
r -> replace
s -> substitute
t -> until befor CHAR (tCHAR) towards right
u -> undo
v -> visual mode
w -> foreward by word (to start of)
x -> delete character under cursor
y -> yank (copy)
z -> :help z (Commands starting with 'z')

A -> append to end of line
B -> backward by word (like b)
C -> change to end of line
D -> delete to end of line
E -> forward by word (to end of)
F -> move to character towards left (FCHAR)
G -> goto line
H -> cursor to top line of screen
I -> insert to beginning of line
J -> join lines
K -> :help K
L -> cursor to low line of screen
M -> cursor to middle line of screen
N -> search again in opposite direction
O -> open line above cursor position
P -> paste before / over cursor position
Q -> switch to ex mode
R -> replace mode
S -> delete Line
T -> until after CHAR (tCHAR) towards right
U -> undo all latest changes on one line
V -> visual mode per line
W -> foreward by word (to start of) (like w)
X -> delete characters before the cursor
Y -> yank (copy) line
ZZ -> write if modified and quit (like :x)

. -> repeat last edit command
: -> enter ex command
/ -> search foreward
" -> hold register, benannter copy und delete buffer
0 -> start of line
^ -> first character of line
$ -> end of line
% -> matching bracket
( -> start of sentence
) -> end of sentence
{ -> start of paragraph
} -> end of paragraph
? -> search backward
~ -> toggle case of character
* -> search word under cursor foreward
# -> search word under cursor backward
+ -> line downward, on the first non-blank character
- -> line upward, on the first non-blank character
& -> :s//~/ (repeat last substitute)
>> -> shift line by value of shiftwidth
<< -> unshift line by value of shiftwidth
top
